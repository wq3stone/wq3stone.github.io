# 在计算机网络体系结构中,分层有哪些优点和缺点?分层应该要注意些什么?
优点:①化难为易,分而治之②各层之间是独立的③灵活性好④结构上可分隔开⑤易于实现和维护 ⑥能促进标准化工作
缺点:①分层太多会使各层间接口增多,层间关系复杂,导致系统开销大、效率低②分层太少会导致每一层功能太多、实现复杂,体现不出分层的优越性。
应注意:①根据功能分层,尽量把相似的功能集中在同一层一个层次的功能应比较完整,而不同层的功能应该不同.
②层间接口应尽量简单,选相互联系最少、通过边界交互信息最少的地方建立边界。

# 什么是网络协议?网络协议三要素分别代表什么含义？
协议是为了使两个对等实体之间能够有效通信,就对等实体需要交换什么信息、如何交换等问题制订的规则
①语义:约定需要发出何种控制信息,对这种控制信息要做出何种响应及操作(语义是对控制信息表达意义的规定)
②语法:对通信过程中传输的数据及控制信息的结构式格式进行约定
③同步:数据传输操作次序的约定

# 协议和服务有何区别?有何关系?

协议的实现,保证了能够向上一层提供服务。使用本层服务的实体只能看见服务,而无
法看见下面的协议。下面的协议对上面的实体是透明的.
协议是“水平”的,即协议是控制对等实体之间通信的规则;服务是“垂直”的,即
服务是由下层向上层通过层间接口提供的。

# OSI体系与TCP/IP体系标准的理念有何不同?TCP/IP体系中网络层以下与OSI有什么不同或关系,为什么没有数据链路层和物理层?

OSI体系试图让所有的网络都遵循统一的规则,有同样的体系结构,从而能够互联互通,从事物本身进行设计,保证其兼容互联性.
TCP/IP体系则面对不同的已成型的网络,通过附加网络层、传输层等高层,构建相同的高层通信,让其互联相通,而不是改造其自身结构。
TCP/IP没有对底层通信子网(数据链路层和物理层)进行设计。TCP/IP是用于异构网互连的,这些通信子网是已有的,如各种局域网广城网、城城网等.所以不需要也不能对通信子网重新设计,只设计这些网络的接口即可。

# 分析OSI参考模型与TCP/IP参考模型的异同点。 

(1)OSI 和 TCP/IP 的相同点是二者均采用层次结构，而且都是按功能分层。
(2)OSI 分七层，分别是物理层、数据链路层、网络层、传输层、会话层、表示层和应用层。OSI 在实际中没有得到成功的应用，获得了一些理论研究的成果。
(3)TCP/IP分为四层，分别是网络接口层、网际层、运输层和应用层。TCP/IP已得到广泛的实际应用，成为了事实上的标准。 

# TCP/IP 网络体系结构分为哪几层？FTP协议、IP协议、TCP协议分别属于 TCP/IP 网络体系结构中的哪一层？

TCP/IP 网络体系结构分为四层，分别是网络接口层、网际层、运输层和应用层。FTP 协议是应用层的协议，IP协议属于网际层的协议，TCP协议是运输层的协议。

# 从多个角度分析比较TCP和UDP这两种协议的特点。 

(1)TCP的特点：面向连接、提供可靠传输服务；所有的TCP连接都是全双工、点到点的通信；不支持多播或者广播；提供面向字节流的服务；能实现流量控制、拥塞控制的功能。 
(2)UDP 的特点：面向无连接、提供不可靠传输服务；支持一对一、一对多、多对一和多对多的交互通信；支持多播或广播；提供面向报文的服务；无流量控制、拥塞控制的功能。 

# 数据编码

## 1.不归零编码(NRZ,NonReturnZero)

高电平表示1,低电平表示0
优点:简单
缺点:抗干扰能力差,收发双方需
要借助另外的措施保证发送时钟与
接收时钟的同步.

## 2.曼彻斯特编码(标准以太网)

·前表示该比特的反(原)码,
后表示该比特的原(反)码
[向下跳表示0(或1)向上跳(表示10)]
优点:不需要另外的同步信号
缺点:两个码元表示一比特,编码效率低,
使信道带宽利用率降低,不宜在高速链路使用

## 3.差分曼彻斯特编码(高速宽带网)

☰每个比特开始处有跳变表示0,无变1
·每个码元中间均跳变
优点:比曼彻斯特编码变化少,更适用
于高速传输

## 4.(王道)归零编码(RZ)

·每个码元中间均跳变到0
优点:自同步
缺点:归零占用宽宽,传输效率低

## 5.(王道)反向归零编码(NRZI)

·电平跳变表示0,电平不变表示1

# 奈氏准则

极限码元传输速率(波特率)为2W(W为信道的频率带宽)
极限数据传输速率(比特率)为2Wlog2V(V为每个码元离散电平数目)

# 香农定理

信道的极限数据传输速率=Wlog2(1+S/N)(单位b/s)
S为信道内所传输的信号的平均功率,N为信道内的高斯噪声功率.
S/N为信噪比(无单位)
信噪比(dB)=10lg(S/N)

# CSMA/CD的作用和原理

CS:网络中的站点都具备一种对总线上所传输的信号或载波进行监测的功能。
MA:当总线上的一个站点占用总线发送信号时，所有连接到同一总线上的其他站点都可以通过各自的接收器接听，只不过目标结点和非目标结点处理方法不同
CD:冲突检测，是一种检测或识别冲突的机制

CSMA/CD协议是解决信道争用的问题的。 其工作原理是：先听后发；边发边听；冲突停发；随机延迟后重发。发送前先监听信道是否空闲，若空闲则立即发送；如果信道忙，则继续监听，一旦空闲就立即发送；在发送过程中，仍需继续监听。若监听到冲突，则立即停止发送数据。等待一段随机时间以后，再重新尝试。 

# 网桥的工作原理和特点是什么?网桥和中继器以及以太网交换机有何异同?

网桥工作在数据链路层,它根据帧的目的地址对收到的进行转发。网桥具有过滤帧的作用,当网桥收到一个帧时,并不是向所有的接口都转发此帧,而是先检查此帧的目的地址,再将该帧转发到哪一个接口

④交换机转发速率比网桥快
⑤交换机功能比网桥多
中继器处理的对象是比特,它不能识别数据钼路中的分组;中继器不隔离网段

# 为什么说tcp是面向字节流，udp面向报文？

UDP： 发送方的UDP对应用程序交下来的报文，在添加首部后就向下交付IP层。UDP对由应用层交下来的报文，既不合并，也不拆分，而是保留这些报文的边界。也就是说，无论应用层交给UDP的报文有多长，UDP都会照样发送，即一次发送一个报文。 在接收方的UDP，对IP层交上来的UDP用户数据报，在去除首部后就原封不动的交付给上层的应用进程。也就是说，UDP一次交付一个完整的报文。因此，应用程序必须选择合适大小的报文。若报文过长，UDP把它交付给IP层后，IP层在传送时可能要进行分片处理，这会降低IP层的效率。反之，若报文过短，UDP把它交付给IP层后，会使IP数据报首部的相对长度太大，同样也会降低IP层的效率。
TCP： 不论应用层发送的报文长度如何，到了传输层后，TCP总是把收到的报文看作一串字节流，并且把每一个字节都进行编号。TCP会根据当前网络的拥塞程度和对方接收缓存的大小，决定现在应当发送多长的报文段。TCP关心的是：必须保证每一个字节都正确无误的传输给对方，并不关心传输了多少个报文段，和每个报文段有多少个字节。这就表明了TCP是面向字节流的。

# 简述TCP在数据传输中采用的可靠传输机制有哪些?

差错检验:发送方发送数据前计算核验和,计算出校验位;接收方取出校验位同样计算校验和.
确认应答:应答目的是把接收情况反馈给发送方,以便让发送方重发.
重传:重传是更正传输错误的机制,也是找回丢失数据包和困超时而失效的数据包,
流量控制:动窗口协议,保证不因接收方的缓冲区溢出而丢失数据.
拥塞控制:慢开始算法,拥塞避免算法,快重传与快恢复

# 交换技术

电路交换：整个报文从源点到达终点，不中断，像在一条管道中传输
报文交换：整个报文传输到相邻节点，存储下来，再根据转发表，转发到下一个节点
分组交换：将报文分成多个分组，再将单个分组传送到相邻的节点，存储下来，再根据转发表进行转发到笑一个节点

电路交换(circuit switching）
电路交换要求必须首先在通信双方之间建立连接通道。在连接建立成功之后，双方的通信活动才能开始。通信双方需要传递的信息都是通过已经建立好的连接来进行传递的，并且这个连接也将一直被维持到双方的通信结束。在某次通信活动的整个过程中，该连接将始终占用着连接建立开始时通信系统分配给它的资源。
线路交换的特点是：数据传输可靠、迅速、有序，但线路利用率低、浪费严重，不适合计算机网络。

报文交换(Message Swintching)
报文交换采用"存储-转发"方式进行传送，无需事先建立线路，事后更无需拆除。
优点：线路利用率高、故障的影响小、可以实现多目的报文；
缺点：延迟时间长且不定、对中间节点的要求高、通信不可靠、失序等，不适合计算机网络。

分组交换(Packet Switching)
将用户通信的数据划分成多个更小的等长数据段，在每个数据段的前面加上必要的控制信息作为数据段的首部，每个带有首部的数据段就构成了一个分组。首部指明了该分组发送的地址。
其中分组交换包括：
　　数据报：分组之间可能存在不同的路径，各个分组分别转发；
　　虚电路：在发送方和接收方建立一条逻辑上相连的虚电路，分组沿着虚电路转播。

三、简答题（每题6分，共36分，酌情给分）
1.  简述具有五层协议的网络体系结构各层的主要功能。
1）应用层：直接为用户的应用进程提供服务。   
2）运输层：向两个主机中进程间的通信提供服务。
3）网络层：为分组交换网上的不同主机提供通信服务。
4）数据链路层：将网络层产生的IP数据报组装成帧，在两个相邻节点间的链路上透明地传送以帧为单位的数据。
5）物理层：透明地传送比特流。
2. 简述CSMA/CD协议的要点。
   (1)适配器从网络获得一个分组，加上以太网的首部和尾部，组成以太网帧，放入适配器的缓存，准备发送。
(2)若适配器检测到信道空闲，就发送该帧。若检测到信道忙，则继续检测并等待信道转为空闲，然后发送这个帧。
(3)发送过程中继续检测信道，若一直未检测到碰撞，就把该帧成功发送完毕。若检测到碰撞，则中止发送，并发送人为干扰信号。
(4)在中止发送后，适配器执行指数退避算法，等待r倍的争用期后，返回步骤(2)。
3. 从工作层次、工作原理等方面比较集线器、网桥及路由器的区别。
1)	集线器工作在物理层，以广播方式转发数据。 
2)	网桥工作在数据链路层，以存储转发方式转发帧。
3)	路由器工作在网络层，以存储转发方式转发数据报。
4. 简述UDP协议的特点。
1)	UDP是无连接的。
2)	尽最大努力交付，不保证可靠交付。
3)	没有拥塞控制。
4)	面向报文。
5)	支持一对一、一对多、多对一、多对多的交互通信。
6)	UDP 用户数据报只有8个字节的首部开销。
